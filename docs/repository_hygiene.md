# Repository Hygiene

Use this rule when deciding whether a file belongs in git:

- Commit files that define behavior, document intent, or serve as curated baseline/reference output.
- Ignore files that are generated by local runs or CI and can be recreated on demand.

## Commit

- source code, tests, workflows, migrations, and configuration
- tracked evaluation inputs under `data/eval/`
- curated reference outputs under `reports/`

## Ignore

- runtime-generated output under `artifacts/`
- generated synthetic data under `data/synthetic/experiments/`
- local machine state such as virtual environments, caches, and secrets
- local planning scratch under `docs/superpowers/plans/` and `docs/superpowers/specs/`

## Routine Local Runs

Use `artifacts/local/...` for routine verification commands so generated output stays out of git.

## Curated Baseline Refreshes

Use `reports/` only when intentionally refreshing repository-owned baseline/reference output.
